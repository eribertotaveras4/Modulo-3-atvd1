1. ¿Qué es un Dato?

Un dato es la información más básica y elemental que se almacena en ellas. Puede ser cualquier valor individual, como un número, una palabra, una fecha o cualquier otro tipo de información que se pueda guardar y manipular en una base de datos.

2. ¿Qué es información?

La información es el resultado procesado y significativo que se obtiene a partir de datos organizados y analizados en una base de datos.

3. ¿Diferencia entre dato e información?

  • Dato: Es una unidad mínima de información cruda, sin procesar y sin contexto.

      Ejemplo;

      El número "10", la palabra "perro"

  • Información: Es el resultado de procesar y organizar datos de manera que tengan significado y utilidad

      Ejemplo;
      la frase "hay 10 perros en el parque" es información derivada de los datos (el número "10" y la palabra "perro").

      En pocas palabras la diferencia es que los datos es como parafraseo y la información es lo detallado del dato

    4. ¿Tipos de datos?

    • Datos Numéricos.
    • Datos de Fecha y Hora.
    • Datos Lógicos (Verdaderos o falsos).
    • Datos de textos.

    5. ¿Qué es un modelado de Datos?

    El modelado de datos es el proceso de diseñar la estructura y relaciones de una base de datos para asegurar que los datos estén organizados de manera eficiente y puedan ser fácilmente gestionados y accedidos dentro de un sistema informático.

    6. ¿Tipos de bases de Datos, según:

    1. Variabilidad

      • Estáticas: Datos que cambian poco o no cambian.
      • Dinámicas: Datos que se actualizan constantemente.

  • Contenido

    1. Textuales: Almacenan principalmente texto.
    2. Multimedia: Almacenan datos como imágenes, audio y video.

  • Modelo de base de datos

    1. Relacionales: Organizan datos en tablas con relaciones.
    2. No Relacionales (NoSQL):Utilizan modelos alternativos como documentos o grafos.

    7. ¿Qué es el modelo relacional?

    El modelo relacional organiza los datos en tablas (filas y columnas) y utiliza claves para establecer relaciones entre ellas, asegurando la integridad de los datos.

    8. ¿Describa los elementos del Diagrama entidad Relacional?

    El Diagrama Entidad-Relación (ER) es una herramienta visual que ilustra entidades, sus características (atributos) y las conexiones entre ellas dentro de una base de datos.

    1. Entidades: Objetos del mundo real representados como rectángulos en el diagrama.
    2. Atributos: Características de las entidades que se describen dentro de los rectángulos.
    3. Relaciones: Asociaciones entre entidades representadas por líneas.
    4. Claves Primarias: Atributos que identifican de manera única cada instancia de una entidad, subrayados en el diagrama.
    5. Claves Foráneas: Atributos que establecen relaciones con claves primarias de otras entidades.
    6. Cardinalidad y Participación: Describen cuántas instancias de una entidad pueden estar asociadas con otra entidad a través de una relación.
    7. Restricciones de Integridad: Reglas que garantizan la precisión y consistencia de los datos, como claves primarias y foráneas.

    9. ¿Qué es una entidad?

    Una entidad es un objeto, persona, lugar o concepto del mundo real que puede ser identificado y representado en una base de datos.

    10. ¿Qué es un atributo y sus tipos?

    Un atributo es una característica o propiedad que describe una entidad en una base de datos

    11. ¿Qué es una relación?

    Una relación en el contexto de bases de datos se refiere a la asociación o conexión que existe entre dos o más entidades.

    12. ¿Qué son las llaves primarias, foránea y sus características?

      Llave primaria: Es un atributo o conjunto de atributos que identifican de manera única cada fila en una tabla. No puede contener valores duplicados y garantiza la integridad de los datos.

      Llave foránea: Es un atributo o conjunto de atributos en una tabla que establece una relación con la llave primaria de otra tabla. Sirve para mantener la integridad referencial entre las tablas.

    13. ¿Qué son las reglas del Negocio?

    Las reglas del negocio son directrices o restricciones que definen cómo debería operar una organización o sistema. Estas reglas son específicas para cada empresa y ayudan a guiar el comportamiento y las decisiones dentro de la organización.

    Qué es la cardinalidad o mapeo?

      Cardinalidad: Se refiere a la cantidad de instancias de una entidad que pueden estar asociadas con instancias de otra entidad a través de una relación. Se describe con términos como "uno a uno", "uno a muchos" o "muchos a muchos".

      Mapeo: Es la representación de cómo las entidades en un modelo de datos se relacionan entre sí, indicando cuántas instancias de una entidad pueden estar relacionadas con cuántas instancias de otra entidad.
